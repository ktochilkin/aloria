## Доводки обучения (отложены сознательно)

### 1. B3 — затухание подсказок по мастерству
Подсказки в форме заявки (`OrderTypeHint`) и авто-объяснение отказа не должны
быть вечными: по мере роста мастерства их стоит приглушать.

- **Почему отложено:** конфликтует с текущим opt-in тумблером режима обучения
  (пользователь включил режим → значит хочет подсказки). Нужен продуманный
  сигнал мастерства и правило затухания, а не просто «скрыть».
- **Где:** `lib/features/learning_mode/presentation/order_form_coaching.dart`,
  интеграция в `lib/features/market/presentation/trade_page.dart`.
- **Сигнал мастерства для опоры:** `progressProvider`
  (`lessonsCompleted` / `quizzesPassed`, серверный).

### 2. Объяснение отказа заявки — для всех, не только в режиме обучения
Человеческое объяснение отклонённой заявки (§6.3) сейчас гейтится `learningMode`.
По §6.3 причину отказа стоит показывать всем пользователям.

- **Почему отложено:** в v1 решили показывать только в режиме обучения.
- **Где:** `showOrderRejectionHelp` в `order_form_coaching.dart`; вызовы в
  `trade_page.dart` (синхронно в `_submit` и асинхронно через
  `ref.listen(ordersProvider)`).
- **Нюанс:** развести «подсказку-обучение» (гейт по режиму) и «объяснение
  отказа» (для всех) — это разные вещи.

### 3. Убрать тестовый ReviewItem (артефакт отладки)
Для проверки экрана повторения был форсирован `ReviewItem` по уроку про инфляцию
для портфеля `T00013` (попадает в «due»). Это отладочный артефакт — убрать.

- **Где:** `aloria-api` (сидинг/миграции в `Program.cs` либо прямая запись в БД).

### 4. Сценарные риск-события в симуляторе (для практики риска в обучении)
Чтобы главы про риск окупались не только текстом, симулятор мог бы инсценировать
инфраструктурные/рыночные события на портфеле учащегося: остановка торгов по
бумаге, заморозка позиции, дефолт эмитента, обвал ликвидности.

- **Почему отложено:** 2026-05-26 решили сначала проработать обучающий контент
  без механик. Сейчас рынок — живой мэтчинг TEREX, сценарных событий в движке нет.
- **Пока без механики:** такие риски отрабатываем в уроке как сценарий-решение
  («бумагу заблокировали — что делаешь?»); живую инсценировку закладываем позже.
- **Масштаб:** новая возможность движка рынка (фича), не контент — обсудить по §3.6.
